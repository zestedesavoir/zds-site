    {% blocktrans %} {{ site_name }} est un logiciel libre, construit à l’aide de nombreuses autres technologies libres. {% endblocktrans %}

    {% blocktrans %} Zeste de Savoir applique ses principes d’ouverture jusqu’à son logiciel interne lui-même. Le site internet que vous êtes en train de consulter est libre et open-source, et son développement est communautaire. Oui, vous pouvez contribuer ! {% endblocktrans %}

    {% blocktrans with plural=contents_count|pluralize_fr %} Le cœur du site est développé en Python, à l’aide de Django, un framework simple, mais puissant. Les données sont stockées dans une base MariaDB, à l’exception des contenus. {% endblocktrans %}

    {% blocktrans %} Chaque contenu est stocké dans un dépôt git, ce qui nous permet de facilement versionner ces derniers : les auteurs et autrices peuvent aisément revenir à n’importe quelle version antérieure. Ces contenus sont constitués de fichiers Markdown et d’un fichier coordinateur, un manifest, stockant les métadonnées du contenu et indiquant à quelle section correspond chaque fichier Markdown. {% endblocktrans %}

    {% blocktrans %} L’interface du site est réalisée à l’aide de Sass et de petites touches de JavaScript. jQuery est également utilisé, mais nous sommes en train de le supprimer petit à petit, au profit de JavaScript pur. {% endblocktrans %}

    {% blocktrans %} Pour compiler et optimiser nos fichiers Sass, assembler nos fichiers JavaScript, et minifier l’ensemble, nous utilisons Gulp, un utilitaire en JavaScript permettant d’automatiser du traitement de fichiers, et PostCSS pour rendre automatiquement nos styles compatibles avec tous les navigateurs. {% endblocktrans %}

    {% blocktrans %} Lorsque l’on rédige sur Zeste de Savoir, nous utilisons du Markdown. Derrière, tourne zmarkdown, qui transforme Markdown en HTML et en LaTeX (pour les exports PDF). Il se base sur remark et son arbre abstrait de syntaxe (AST) mdast. Il y a des plans pour migrer vers micromark, le successeur des précédents, mais cela nécessite beaucoup de temps et d’effort. {% endblocktrans %}

    {% blocktrans %} Enfin, pour tester automatiquement tout cela, nous utilisons le framework de tests de Django (basé sur unittest) pour le serveur, et Jest pour zmarkdown. {% endblocktrans %}
